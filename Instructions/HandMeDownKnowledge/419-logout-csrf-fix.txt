TITLE: 419 Page Expired on Logout & Login — CSRF Token Mismatch (Static Meta Tag Conflict)
DATE: 2026-03-09

---

## SESSION 1 — 2026-03-06: Logout Double-Submit Race Condition

### GOAL
Logout must complete cleanly without 419 errors across all user roles.

### PROBLEM
Logging out intermittently produced HTTP 419. Log pattern: ~513ms request then ~0.36ms second
request to /logout. Second request hit already-invalidated session — CSRF token gone.

### KEY DISCOVERIES
- LogoutButton.jsx had no double-submit guard; rapid clicks could queue multiple Inertia POSTs.
- First request: Auth::logout() + invalidate() + regenerateToken() — session dead.
- Second request (0.36ms): POST with stale CSRF token → 419.
- AuthController::logout(), routes, session config were all correct.

### DECISIONS
- Added `useRef(false)` mutex flag in LogoutButton. `onFinish` resets it.
- Used `useRef` not `useState` — no re-render needed, purely a guard.
- No backend changes.

### FILES MODIFIED
- resources/js/Components/Common/LogoutButton.jsx

### STATUS: RESOLVED

---

## SESSION 2 — 2026-03-09: 419 on Logout AND Login — Static CSRF Meta Tag Poisoning Axios

### INPUT
- 419 on POST /logout (~514ms response — full round trip of a rejected request, not a fast second hit).
- After adding TokenMismatchException handler, 419 moved to POST /login as well, including incognito.

### PROCESS

**What was checked:**
- APP_URL = http://localhost ✅ (already fixed in previous session)
- SESSION_DRIVER = file ✅
- AuthController::logout() is correct ✅
- LogoutButton.jsx uses `router.post` with processing guard ✅
- app.blade.php had `<meta name="csrf-token" content="{{ csrf_token() }}">` ⚠️
- bootstrap.js manually pinned `X-CSRF-TOKEN` from that meta tag onto all axios requests ⚠️
- storage/framework/sessions/ contained 3 stale file-session entries from http://127.0.0.1:8000 (corpse sessions, not the direct cause)

**Root cause:**
Inertia.js does NOT use axios headers for its own CSRF handling. It reads the `XSRF-TOKEN`
cookie set by Laravel's VerifyCsrfToken middleware and sends it as `X-XSRF-TOKEN`.
The static `<meta name="csrf-token">` tag gives a token that is only valid for the initial
page load. bootstrap.js was pinning that stale token to axios, which conflicted and caused
all Inertia POSTs (login, logout, any form) to 419.
The Inertia docs explicitly warn: omit the csrf-token meta tag when using Inertia.

### OUTPUT

**Fix 1 — Global TokenMismatchException handler (bootstrap/app.php):**
Any 419 anywhere in the app now redirects cleanly to login instead of showing a raw error.
This is the single upstream enforcement point per project rules.

**Fix 2 — Removed `<meta name="csrf-token">` from app.blade.php.**
Inertia manages CSRF via the XSRF-TOKEN cookie automatically; the meta tag is not needed
and actively harmful when bootstrap.js reads it.

**Fix 3 — Removed manual X-CSRF-TOKEN axios header from bootstrap.js.**
Axios reads XSRF-TOKEN cookie natively and sends X-XSRF-TOKEN on POSTs automatically.
No manual wiring is needed or correct.

### KEY DISCOVERIES
- Inertia CSRF = XSRF-TOKEN cookie → X-XSRF-TOKEN header. Not the meta tag. Not axios defaults.
- Static meta tag token goes stale after the first request that regenerates the session token.
- The 419 on logout was never a logout-specific bug — it was the same meta tag CSRF issue
  surfacing on every POST. The Session 1 fix (double-submit guard) masked it on logout
  by preventing a second attempt, but the root cause was always the poisoned axios header.
- Stale session files in storage/framework/sessions/ referencing 127.0.0.1:8000 are
  harmless leftovers. They do not cause 419. Clean them out manually if desired.

### DECISIONS
- TokenMismatchException handler added to bootstrap/app.php withExceptions() block.
- Meta tag removed. Bootstrap.js manual CSRF wiring removed.
- No changes to AuthController, routes, LogoutButton, or session config.

### ATTEMPTED SOLUTIONS / DEAD ENDS
- Clearing config/cache + hard browser refresh: did not resolve (correct step but not the fix).
- Suspected stale cookies in browser: ruled out — reproduced in incognito with no cookies.
- Suspected stale session files on disk: not the cause — Inertia creates fresh sessions per visit.

### FILES MODIFIED
- bootstrap/app.php (TokenMismatchException handler)
- resources/views/app.blade.php (removed csrf-token meta tag)
- resources/js/bootstrap.js (removed manual X-CSRF-TOKEN axios header)

### POST-FIX STEPS REQUIRED
```
php artisan config:clear
php artisan cache:clear
npm run build  (or npm run dev)
```

### STATUS: RESOLVED
