TITLE: 419 Page Expired on Login — APP_URL Mismatch Corrupting CSRF/Session Cookies
DATE: 2026-03-06

GOAL
----
Login POST must complete without 419 across all roles.

PROBLEM DEFINITION
------------------
HTTP 419 on POST /login. Log shows GET /login (~513ms) followed immediately by POST /login (~513ms) → 419.
Also observed: /logout completing in ~0.3ms (session already dead before logout fires).

KEY DISCOVERIES
---------------
- APP_URL was set to http://127.0.0.1:8000 in .env
- Actual XAMPP access URL is http://localhost (port 80, no port suffix)
- Laravel uses APP_URL to construct session cookie domain. Mismatch means the browser
  sends the GET /login response (with CSRF token) but the session cookie is scoped to
  127.0.0.1:8000 — the subsequent POST /login from localhost has no matching session,
  so CSRF validation fails → 419.
- SESSION_DRIVER=file in .env (overrides config/session.php default of 'database'). This is fine.
- Login.jsx and AuthController::login() are structurally correct and not the cause.

DECISIONS & RATIONALE
----------------------
- Changed APP_URL to http://localhost in .env — must exactly match the browser-facing URL.
- Required follow-up: php artisan config:clear && php artisan cache:clear
- Required follow-up: hard browser refresh (Ctrl+Shift+R) to purge stale cookies

ATTEMPTED SOLUTIONS / FAILURES
--------------------------------
- None failed. Root cause identified from .env vs actual access URL mismatch.

CURRENT STATUS
--------------
RESOLVED. .env updated. Config/cache must be cleared before testing.

FILES MODIFIED
--------------
